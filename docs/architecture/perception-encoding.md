# Perception Encoding

사용자의 추상적인 시각 선택을 DS 토큰으로 번역하는 매핑 시스템.

## 개념

사용자는 디자인 용어를 모른다. 하지만 감각적인 선택은 할 수 있다.
Perception Encoding은 그 선택을 해석해서 토큰 값으로 변환한다.

```
추상 카드 선택 → Perception Encoding → Design Tokens
```

## 구조

### 입력 (카드 선택)
- 사용자에게 추상적인 비주얼 카드 쌍을 제시
- 디자인 시스템이라는 단어가 연상되지 않는 순수한 감각적 이미지
- 여러 쌍을 선택하면서 점진적으로 정보 수집

### 매핑 레이어
각 카드 선택은 하나 이상의 토큰 파라미터에 영향을 준다.

| 지각 축 | 영향을 받는 토큰 |
|--------|----------------|
| 긴장 ↔ 여유 | spacing, density |
| 날카로움 ↔ 부드러움 | border-radius |
| 무거움 ↔ 가벼움 | shadow, font-weight |
| 조용함 ↔ 시끄러움 | color saturation, contrast |
| 빠름 ↔ 느림 | transition duration, easing |

### 출력 (Design Tokens)
- W3C DTCG 포맷 JSON
- 선택이 충돌할 경우 AI가 보정

## 카드 설계 원칙

1. 카드 한 쌍은 여러 파라미터를 동시에 커버할 수 있다
2. 선택지는 추상적일수록 좋다 — 도형, 질감, 밀도, 움직임
3. 5-7쌍으로 핵심 토큰 전체를 커버하는 것을 목표로 한다
4. 선택할 때마다 DS 사이트가 실시간으로 반응한다
